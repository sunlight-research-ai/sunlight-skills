## Findings
- Make's ability to use a Figma design or image as a reference supports the claim that Figma can steer AI generation with existing product context. [SRC_003]
- MCP support for IDEs, coding agents, browser models, and Figma Make supports the claim that Figma wants AI work to carry design/code context across tools. [SRC_005] [SRC_006]
- Code visibility inside Make is strategically important because AI agents can reason from underlying code rather than only from rendered screenshots or prototypes. [SRC_006]
- Practitioner evidence suggests Figma MCP workflows still require detailed context and iteration, which argues against treating one-shot UI generation as the durable moat. [SRC_010]
